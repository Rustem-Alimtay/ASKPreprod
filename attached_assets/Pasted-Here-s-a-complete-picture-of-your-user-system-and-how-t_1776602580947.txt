The 5 Roles & What They Can Do
Role	Main Job	Specific Powers
ADMIN	Full control	Everything — including all sign-offs, user management, settings, audit logs, deleting invoices/stables/boxes
LIVERY_ADMIN	Day-to-day operations	Create/edit horses (one at a time, no Excel import), agreements, billing elements, generate invoices (after both sign-offs), rollback invoices

STORES	Inventory sign-off	Toggle the Store approval button on the To Invoice page

What's hidden per role (UI-side)
Administration sidebar section (Users, Settings, Audit Logs) → only visible to ADMIN
Excel "Import" buttons on Horses/Boxes/Customers/Stables/Items → only ADMIN
"New Horse" button → ADMIN + LIVERY_ADMIN
Vet sign-off button → only ADMIN + VETERINARY can click it
Store sign-off button → only ADMIN + STORES can click it
"Send to NetSuite" / "Generate SO" on invoices → only ADMIN + FINANCE
"Rollback invoice" → only ADMIN + LIVERY_ADMIN
"Delete invoice" → only ADMIN

How to Test Each Role Manually
You have two options. I'd recommend Option A for ease.

Option A — Create test accounts with passwords (recommended):
Log in as admin → go to Administration → Users → create one test user per role:

test_livery → LIVERY_ADMIN
test_vet → VETERINARY
test_stores → STORES
test_finance → FINANCE
Set a known password for each (e.g. test123). Then log in/out to switch between them.

Option B — Temporarily change admin's role in the Users page to test each role's UI, then change it back. Faster but you can't test cross-role workflows (e.g., Vet approves, then Livery Admin generates).

Manual Test Scenarios
Scenario 1 — End-to-end invoice workflow (multi-role)
As LIVERY_ADMIN: Go to Billing Elements → add 2-3 adhoc items for a customer's horse with a transaction date in the current month → Save.
As LIVERY_ADMIN: Go to To Invoice → select the billing month → confirm both sign-off buttons say "Vet Validation Pending" / "Store Validation Pending" → confirm Generate Invoice is disabled. Try clicking the Vet/Store sign-off buttons — they should be disabled (you're not Vet/Stores).
As VETERINARY: Go to To Invoice → click "Vet Validation Pending" → it turns green and shows "Vet Approved" with your name. Try clicking the Stores button — should be disabled.
As STORES: Click "Store Validation Pending" → turns blue, "Store Approved".
As LIVERY_ADMIN: Go back to To Invoice → Generate Invoice button should now be enabled → select line items → Generate → customer disappears from To Invoice; new invoice appears on Invoices page with status APPROVED.
As FINANCE: Go to Invoices → click "Generate SO" then "Send to NetSuite" on the new invoice. Verify the buttons exist and work.
As LIVERY_ADMIN: Try to rollback the invoice (should be allowed). Try to delete it (should be hidden — only ADMIN sees that).
As VETERINARY: Try to go back and toggle the approvals for that customer/month — should be locked because the invoice exists.
Scenario 2 — Permission boundaries

Direct URL test: As any non-admin, try /admin/users, /admin/settings, /admin/audit-logs — all should redirect.
